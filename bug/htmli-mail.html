HTML-I

HTML Injection at Email triggering  

Date: 26/7/2023

In the domain example.com, There was a website that had a form. The form took inputs such as first name, last name, email address, and sent an email to the user. I filled out the form and intercepted the request using Burp Suite.

Burp Request 
    
POST /subscribe/form HTTP/1.1
Host: www.example.com
Cookie:
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:120.0) Gecko/20100101 Firefox/120.0
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/avif,image/webp,*/*;q=0.8
Accept-Language: en-US,en;q=0.8,en-US;q=0.5,en;q=0.3
Accept-Encoding: gzip, deflate, br
Content-Type: application/x-www-form-urlencoded
Content-Length: 30
Origin: https://www.example.com
Referer: https://www.example.com/subscribe/form
Upgrade-Insecure-Requests: 1
Sec-Fetch-Dest: document
Sec-Fetch-Mode: navigate
Sec-Fetch-Site: same-origin
Sec-Fetch-User: ?1
Te: trailers
Connection: close
        
emailAddress=test%40mail.com&firstname=John&lastname=Doe

After receiving the email, it addressed the recipient as "Hi John Doe," highlighting both the last name and first name. Consequently, I made an attempt to inject HTML form code, resulting in the highlighting of my HTML code in the email.


Firstname=%3Cform%20%3E%3Cinput%3E%3C%2Finput%3E%3Cinput%20type%3D%22submit%22%2F%3E%3C%2Fform%3E




Due to insufficient sanitization, the form was vulnerable to HTML injection (HTMLi). Malicious attackers could exploit this vulnerability to launch phishing campaigns and potentially extract sensitive data from unsuspecting users. HTML injection occurs when user input is not properly sanitized, allowing attackers to inject malicious HTML code into the web page. This injected code can manipulate the appearance and behavior of the page, leading to unauthorized actions or the theft of confidential information
